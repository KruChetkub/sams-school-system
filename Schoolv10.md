# Development Plan: School Attendance Management System (SAMS) - Version 10
## สรุปการอัปเดตล่าสุด + Baseline สำหรับพัฒนาต่อจาก V9 (โฟกัสระบบเยี่ยมบ้าน)

เอกสารนี้สรุปงานที่พัฒนาเพิ่มเติมในรอบล่าสุด เพื่อใช้เป็น baseline ของ V10 โดยเน้นการพัฒนาระบบ "เยี่ยมบ้าน (Home Visit)" อย่างเต็มรูปแบบ ทั้งด้าน UI/UX, การจัดการแผนที่ (Map & Routing), และการเชื่อมต่อฐานข้อมูลที่ลื่นไหล

---

## 1) สถานะความสำเร็จในรอบล่าสุด (V10 Completed)

### 1.1 แผนที่และการระบุพิกัด (Visit Map & Routing)
- [x] อัปเดตพิกัดศูนย์กลางเป็น **โรงเรียนเชียงของวิทยาคม** (Lat: 20.2445000, Lng: 100.4125000)
- [x] โหลดและแสดงเส้นขอบเขตอำเภอเชียงของผ่าน GeoJSON
- [x] **คำนวณระยะทางอัตโนมัติ (OSRM API):** ระบบสามารถดึงพิกัดบ้านนักเรียนแล้วคำนวณเส้นทางถนนจริง (Driving Route) ไปยังโรงเรียนได้
- [x] แสดงระยะทางเป็น Floating Overlay บนแผนที่อย่างชัดเจน
- [x] **Silent Sync:** หากแผนที่คำนวณระยะทางสำเร็จ จะมีการเช็คฐานข้อมูล ถ้าค่าระยะทางเปลี่ยนหรือยังไม่มี ระบบจะอัปเดตลงตาราง `home_visits` อัตโนมัติและ Refresh ตารางประวัติทันที (React Query Invalidation)

*ไฟล์ที่แก้:* `src/components/homevisit/VisitMap.tsx`, `src/services/homevisit/visitService.ts`

### 1.2 หน้าประวัติการเยี่ยมบ้าน (Visit Dashboard)
- [x] ปรับหน้ารวมประวัติ (Visit History) จาก List ธรรมดา ให้เป็น **Data Table** เต็มรูปแบบ 
- [x] คอลัมน์ประกอบด้วย: ลำดับที่, คำนำหน้า ชื่อ สกุล, วันที่เยี่ยมบ้าน (ภาษาไทย), ระยะทางถึงโรงเรียน, และปุ่มเปิดดูรายละเอียด
- [x] แสดง Modal รายละเอียดการเยี่ยมบ้าน (ข้อมูลเด็ก, วันที่แบบไทย, ผลประเมินความเสี่ยง)
- [x] เพิ่มปุ่ม **"ดูเส้นทางไปโรงเรียน"** ใน Modal ซึ่งเมื่อกดปุ๊บ ระบบจะ:
  1. ปิดหน้าต่าง Modal
  2. เลื่อน (Scroll) หน้าจอกลับขึ้นไปที่แผนที่ด้านบน
  3. วาดเส้นทาง (Polyline) สีฟ้าบนแผนที่ และโชว์ตัวเลขระยะทาง

*ไฟล์ที่แก้:* `src/pages/homevisit/Dashboard.tsx`

### 1.3 หน้ารายชื่อนักเรียน (Students List & Admin View)
- [x] ปรับหน้า UI/UX ของรายชื่อนักเรียน ให้แยกความแตกต่างชัดเจน:
  - **เยี่ยมแล้ว (COMPLETED):** การ์ดพื้นหลังไล่สีเขียว (Gradient Emerald), ปุ่มกดเป็นสีเขียวเข้มเขียนว่า "ตรวจสอบ / แก้ไข"
  - **ยังไม่ได้เยี่ยม:** การ์ดสีขาวขอบเทาดูสะอาดตา, ปุ่มสีน้ำเงินอ่อนเขียนว่า "บันทึกการเยี่ยมบ้าน"
- [x] ในมุมมองแอดมิน: โชว์การ์ดสรุปครูที่ปรึกษารายห้อง และใช้สี Gradient แบบ Deterministic (ตาม ID ครู)

*ไฟล์ที่แก้:* `src/pages/homevisit/StudentsList.tsx`

### 1.4 หน้าบันทึกแบบฟอร์ม (Visit Form)
- [x] เปลี่ยน Date Picker เดิมเป็นแบบ Dropdown 3 ช่อง (วัน, เดือน, ปี พ.ศ. แบบไทย)
- [x] คำนวณปี พ.ศ. เป็น ค.ศ. คืนให้ฐานข้อมูล (`YYYY-MM-DD`) ได้ถูกต้อง
- [x] ดึงพิกัด GPS พร้อม **คำนวณระยะทางไปโรงเรียนทันที** แล้วโชว์ตัวเลขที่หน้าฟอร์มก่อนกดบันทึก
- [x] **แทนที่ alert() และ window.confirm()** แบบดั้งเดิมด้วย Custom UI Modals ทั้งหมด (Success, Error, GPS Confirm)
- [x] แก้ไขบั๊ก Supabase (null id constraint) เวลาสร้างแบบประเมินใหม่ให้บันทึกได้สมบูรณ์

*ไฟล์ที่แก้:* `src/pages/homevisit/VisitForm.tsx`

### 1.5 ระบบจัดการโฟลเดอร์รูปภาพเยี่ยมบ้าน (Supabase Storage)
- [x] สร้างโครงสร้างโฟลเดอร์อัตโนมัติ โดยอ้างอิงจาก `student_code`
- [x] ชื่อไฟล์จะถูกตั้งตามแพทเทิร์น `homevisit_{studentCode}_{timestamp}.jpg` เพื่อให้เป็นระเบียบและค้นหาง่ายใน Storage `home_visit_photos`

*ไฟล์ที่แก้:* `src/services/homevisit/visitService.ts`, `src/pages/homevisit/VisitForm.tsx`

### 1.6 การปรับปรุง UI/UX ระดับ Premium (Login & Portal)
- [x] **หน้า Portal:** อัปเกรดพื้นหลังเป็นแอนิเมชัน Lottie (`A.json`) ที่มีความเคลื่อนไหวทันสมัย
  - ปรับให้แอนิเมชันแสดงผลเต็มหน้าจอโดยไม่มีขอบ (Skip Intro Loop & Scale)
  - นำเทคนิค **Glassmorphism (กระจกฝ้า)** มาใช้กับแถบเมนูด้านบน เพื่อให้ตัวหนังสือโดดเด่นและอ่านง่าย แม้พื้นหลังจะเคลื่อนไหว
  - ปรับเลย์เอาต์ ยกการ์ดเมนูหลัก (SAMS, เยี่ยมบ้าน) ขึ้นมาตรงกลางหน้าจอ ลดทอนสิ่งรบกวนสายตา
- [x] **หน้า Login:** อัปเกรดพื้นหลังเป็น Lottie (`purple.json`) 
  - ตั้งค่า `preserveAspectRatio` ให้พื้นหลัง Responsive ยืดหดเข้ากับทุกสัดส่วนหน้าจอแบบ Cover พอดีเป๊ะ
  - ผสานพื้นหลังเข้ากับเอฟเฟกต์เมาส์ขยับ (Lightning Sparkles) ได้อย่างสมบูรณ์
  - แก้ไขปัญหาทางเทคนิค (React Duplicate Key Warning) ในการวาดเส้นเอฟเฟกต์ให้เสถียร 100%

*ไฟล์ที่แก้:* `src/pages/portal/Portal.tsx`, `src/pages/Login.tsx`

## 2) Database Schema Changes & Requirements (V10)

เพื่อรองรับฟีเจอร์ใหม่ในรอบนี้ มีการปรับโครงสร้าง/ความต้องการของฐานข้อมูลดังนี้:

### 2.1 Storage Bucket
- ต้องสร้าง Bucket ชื่อ: `home_visit_photos`
- ต้องตั้งสถานะเป็น: **Public**
- ต้องมี Policies (RLS) ที่อนุญาตให้ทำ `SELECT, INSERT, UPDATE, DELETE` เพื่อให้ระบบอัปโหลดไฟล์ได้

### 2.2 Table: `home_visits`
เพิ่มคอลัมน์ใหม่สำหรับเก็บระยะทางจากบ้านนักเรียนถึงโรงเรียน
```sql
ALTER TABLE public.home_visits 
ADD COLUMN IF NOT EXISTS distance_to_school numeric;
```
