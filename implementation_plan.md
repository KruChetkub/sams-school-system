# แผนการพัฒนา: ระบบสิทธิ์ผู้ดูแลระบบเสริม (is_admin_allowed) สำหรับบทบาทครู

แผนงานนี้จัดทำเพื่อรองรับความต้องการที่ **ผู้ใช้บางคนต้องมีสิทธิ์เข้าถึง 2 สิทธิ์ควบคู่กัน (เช่น มีบทบาทหลักเป็นครู `TEACHER` เพื่อให้มีประวัติครูและสามารถเช็คชื่อนักเรียนได้ แต่ก็สามารถได้รับสิทธิ์แอดมินเสริมเพื่อควบคุมระบบดูแลช่วยเหลือได้ด้วย)**

---

## 🛠️ ขั้นตอนและสิ่งที่ต้องดำเนินการ

### 1. ปรับปรุงฐานข้อมูล (Supabase SQL)
เราจะแก้ไขไฟล์ [supabase_user_management_rpc.sql](file:///d:/sams-school-system-main/frontend/supabase_user_management_rpc.sql) เพื่อเพิ่มการตั้งค่านี้:
- เพิ่มคอลัมน์ `is_admin_allowed` ในตาราง `public.users` (เป็น Boolean เริ่มต้นเป็น `false`)
- อัปเดตฟังก์ชันความปลอดภัยในการเช็คสิทธิ์ `public.is_admin()` และ `public.is_admin_or_executive()` ให้ยอมรับกรณีที่ `is_admin_allowed = true` ให้เข้าถึงฐานข้อมูลในฐานะผู้ดูแลระบบได้ด้วย
- อัปเดตฟังก์ชัน RPC `admin_create_user` และ `admin_update_user` เพื่อให้สามารถบันทึกค่า `is_admin_allowed` จากหน้าเว็บได้

---

### 2. ปรับปรุงระดับสิทธิ์ฝั่ง React (authStore.ts)
เราจะดึงค่า `is_admin_allowed` จากฐานข้อมูลมาประมวลผลร่วมกับบทบาทหลักของผู้ใช้

#### [MODIFY] [authStore.ts](file:///d:/sams-school-system-main/frontend/src/store/authStore.ts)
- เพิ่ม `isAdminAllowed` ลงใน Zustand State และดึงค่ามาพร้อมกับการ Query Role ตอนเข้าสู่ระบบ
- ปรับแต่ง `useAuth` helper ให้ส่งคืน `isAdmin = true` หากผู้ใช้มีบทบาทเป็น `ADMIN` หรือ `SUPER_ADMIN` หรือมี `isAdminAllowed = true`

---

### 3. ปรับปรุงระดับ Service (userService.ts)
เราจะปรับปรุงฟังก์ชันของบริการให้ส่งอาร์กิวเมนต์สิทธิ์เสริมไปยังฐานข้อมูล

#### [MODIFY] [userService.ts](file:///d:/sams-school-system-main/frontend/src/services/userService.ts)
- อัปเดตอินเตอร์เฟซ `AppUser` ให้ครอบคลุมฟิลด์ `is_admin_allowed?: boolean`
- ปรับฟังก์ชัน `adminCreateUser` และ `adminUpdateUser` ให้รองรับพารามิเตอร์ `isAdminAllowed` และเรียกใช้ RPC เวอร์ชั่นใหม่

---

### 4. ปรับปรุงหน้า UI ตั้งค่าผู้ใช้งาน (Settings.tsx)
เราจะเพิ่มส่วนการตั้งค่าปุ่มเปิด-ปิดสิทธิ์ผู้ดูแลระบบเสริมในหน้าจอตั้งค่าผู้ใช้

#### [MODIFY] [Settings.tsx](file:///d:/sams-school-system-main/frontend/src/pages/Settings.tsx)
- เพิ่มปุ่ม Toggle (สวิตช์เปิด-ปิด) หรือ Checkbox **"สิทธิ์ผู้ดูแลระบบเสริม (Admin Access)"** ใน:
  - **Modal เพิ่มผู้ใช้งานใหม่**
  - **Modal แก้ไขข้อมูลผู้ใช้งาน**
- แสดงสถานะสิทธิ์เสริมในตารางรายชื่อผู้ใช้งานด้วยไอคอนหรือป้ายกำกับขนาดเล็ก เช่น `+ Admin` ข้างสิทธิ์หลัก

## 📋 แผนการตรวจสอบและทดสอบ (Verification Plan)

### การทดสอบด้วยตนเอง (Manual Verification)
1. **รัน SQL Script อัปเกรดใน Supabase SQL Editor**
2. **สร้างผู้ใช้บทบาทครูที่มีสิทธิ์เสริมเป็นแอดมิน:**
   - สร้างผู้ใช้ใหม่ เลือกบทบาท `TEACHER` และติ๊กเปิดใช้ "สิทธิ์ผู้ดูแลระบบเสริม"
   - ล็อกอินด้วยบัญชีนี้ -> ตรวจสอบว่าสามารถเข้าถึงเมนูแดชบอร์ดแอดมิน และหน้าจัดการปีการศึกษา/ข้อมูลหลักได้เหมือนกับบทบาทแอดมินทั่วไป
3. **ตรวจสอบหน้า Teachers:**
   - เข้าหน้า **Teachers** -> ตรวจสอบว่ามีแถวของครูคนนี้ให้กรอกประวัติเพิ่มเติมได้ตามปกติ (เพราะมีบทบาทหลักเป็น `TEACHER`)
4. **ลบสิทธิ์แอดมินเสริม:**
   - เข้าด้วยบทบาท `SUPER_ADMIN` ไปที่ Settings
   - แก้ไขข้อมูลครูคนเดิม ติ๊กปิด "สิทธิ์ผู้ดูแลระบบเสริม" แล้วกดบันทึก
   - บัญชีดังกล่าวเมื่อล็อกอินใหม่ จะเข้าหน้าแอดมินไม่ได้อีกต่อไป แต่ยังคงเข้าสู่ระบบในฐานะครูทั่วไปและทำงานประวัติครูได้ตามเดิม
