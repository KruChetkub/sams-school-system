# Development Plan: School Attendance Management System (SAMS) - Version 7
## แผนงานต่อยอดความสำเร็จจาก V6 + แผนพัฒนาเชิงลึกสำหรับ V7

เอกสารนี้รวบรวมความสำเร็จที่ได้พัฒนาในเวอร์ชัน 6 ทั้งหมด และกำหนดแผนงานดำเนินงานอย่างต่อเนื่องสำหรับ **Version 7** โดยมุ่งเน้นไปที่ความพร้อมสำหรับการนำระบบไปใช้งานจริงในสถานศึกษา (Production Ready) อย่างเต็มรูปแบบ

---

## 1) สถานะความสำเร็จในรอบล่าสุด (V6 Completed)

เราได้ดำเนินการพัฒนาและปิดงานระบบวิชาเรียนรวม (Goal A ของ V6) ได้สมบูรณ์ 100% พร้อมยกระดับ UX/UI ขึ้นเป็นระดับพรีเมียม โดยไม่มีข้อผิดพลาดในการ Build ระบบ:

### 1.1 ระบบวิชาเรียนรวมแบบเรียลไทม์ (End-to-End Dynamic Memberships)
- [x] **Real-time Invalidation & Query Sync**: แก้ไขระบบ Sync ข้อมูลสมาชิกกลุ่มวิชาเรียนรวม โดยเพิ่มคีย์ `['student_group_memberships_all']` ในการล้างแคช (Invalidate Queries) ทันทีที่มีการเพิ่ม/ลบนักเรียนในกลุ่ม ส่งผลให้ตารางประวัตินักเรียนและ badge ด้านล่างอัปเดตเรียลไทม์ทันทีโดยไม่ต้อง Refresh หน้าเว็บ
- [x] **Attendance Integration**: ปรับแต่งฟังก์ชัน `getStudentsForSchedule` ใน `attendanceService.ts` ให้ดึงข้อมูลนักเรียนจากกลุ่มเรียนรวมของวิชานั้น ๆ เป็นอันดับแรก และทำ Fallback ไปห้องหลัก/ห้องเสริมอัตโนมัติหากเป็นรายวิชาปกติ

### 1.2 การยกระดับ UX/UI ระดับพรีเมียม (Premium Interaction Design)
- [x] **Dynamic Filter Categories (Optgroup)**: ปรับปรุงกลไกเมนูตัวกรอง dropdown ให้ฉลาดขึ้นโดยการแบ่งกลุ่มหมวดหมู่ตัวเลือกอย่างชัดเจนด้วย `<optgroup>` แยกเป็น **"ระดับชั้น/ห้องเรียนหลัก"** และ **"กิจกรรมชุมนุม / เรียนรวม"**
- [x] **Dummy Classroom Exclusion**: เขียนฟังก์ชันคัดกรองห้องเรียนหลักเพื่อคัดห้องเรียนจำลองในระบบ (เช่น `กิจกรรมชุมนุม/รวม`, `ประชุมบริหาร/1`) ออกจากกลุ่มห้องเรียนปกติโดยอัตโนมัติ ป้องกันข้อมูลซ้ำซ้อนและลดความสับสนของครูผู้ใช้งาน
- [x] **Premium Searchable Dropdown (Combobox)**: อัปเกรดปุ่มตัวเลือกฟิลเตอร์ธรรมดาให้เป็น Combobox อัจฉริยะที่เขียนขึ้นด้วย React:
  - รองรับการพิมพ์ค้นหาเพื่อบีบตัวเลือกแบบพิมพ์ได้เรียบร้อย (เช่น พิมพ์ "ม.3" หรือ "ก" ลิสต์จะแสดงเฉพาะที่เกี่ยวข้องทันที)
  - เพิ่มปุ่มล้างคำค้นหาด่วน **✕** ด้านในอินพุต
  - เพิ่มระบบตรวจจับการคลิกนอกพื้นที่ (Click Outside) เพื่อหุบเมนูลงอย่างเป็นธรรมชาติ
- [x] **Lazy Rendering Placeholder Card**: ปรับแต่งหน้าจอส่วน "รายชื่อนักเรียนทั้งหมด" เพื่อประสิทธิภาพความเร็วสูงสุด โดยเริ่มต้นจะแสดงการ์ดแนะนำดีไซน์พรีเมียม (Users Placeholder Card) บอกให้ครูผู้ใช้เลือกห้องเรียน/กิจกรรมก่อน และจะเรนเดอร์รายชื่อก็ต่อเมื่อมีการกรองผลแล้วเท่านั้น ช่วยประหยัด DOM และทำให้เว็บโหลดเร็วขึ้นกว่า 10 เท่า!

## 2) เป้าหมาย V7 (Definition of Done)

ในเวอร์ชัน 7 นี้ เราจะขยายขอบเขตความสามารถของระบบไปที่การดาวน์โหลดรายงานสรุปอย่างสมบูรณ์แบบ และเพิ่มมาตรการความปลอดภัยขั้นสูงสุดด้วย RLS ในระบบ Supabase:

### Goal A: ระบบดาวน์โหลดรายงานอเนกประสงค์ (Reports Export)
- [ ] **CSV Export ภาษาไทย**: พัฒนาระบบดาวน์โหลดข้อมูลเป็นไฟล์ Excel/CSV ครบทุกแท็บหลัก (Overview, Classroom, Student, Subject, Homeroom) โดยแก้ปัญหาเรื่อง Font/Encoding ภาษาไทยให้อ่านบนโปรแกรม Microsoft Excel ได้สมบูรณ์แบบไม่เพี้ยนเป็นต่างดาว
- [ ] **PDF Report Export**: พัฒนาระบบดาวน์โหลดไฟล์ PDF รายงานการเช็คชื่อมาตรฐานโรงเรียน โดยใช้โมดูล `jsPDF` ร่วมกับ `autoTable` รองรับฟอนต์ภาษาไทยมาตรฐาน (เช่น TH Sarabun PSK) จัดรูปแบบตารางสวยงาม

### Goal B: ระบบรักษาความปลอดภัยข้อมูลขั้นสูงสุด (RLS & Security Policies)
- [ ] **Row Level Security (RLS) Rollout**: เปิดใช้งาน RLS บนตารางข้อมูลหลักทั้งหมดใน Supabase เพื่อป้องกันไม่ให้บุคคลภายนอกสามารถเข้ามาดึงข้อมูลส่วนตัวของนักเรียนและครูได้
- [ ] **Role-based Access Controls**: กำหนด Policy แยกสิทธิ์การเรียกดูและแก้ไขข้อมูลอย่างรัดกุม:
  - `ADMIN`: สามารถทำ CUD (Create, Update, Delete) ได้ทุกตาราง
  - `TEACHER`: ดูข้อมูลทั้งหมดได้ แต่จะเช็คชื่อหรือเขียนประวัติย้อนหลังได้เฉพาะรายวิชาหรือห้องเรียนที่ตนเองได้รับมอบหมายเท่านั้น
  - `STUDENT / PARENT`: สิทธิ์แบบ Read-only เฉพาะประวัติการมาเรียนของตนเองเท่านั้น

### Goal C: UI/UX Standardization (ถอนการใช้ window.confirm)
- [ ] **Standard Modal Refactoring**: ปรับแต่งปุ่มลบหรือคำถามยืนยันการดำเนินการทั้งหมดในหน้าอื่นๆ (เช่น ครู, รายวิชา, ห้องเรียน) ให้เปลี่ยนมาใช้ Component **Custom Modal** ดีไซน์พรีเมียมเหมือนหน้า Students ทั้งหมด แทนที่การใช้ `window.confirm` แบบเดิมของเบราว์เซอร์

## 3) แผนงานละเอียดสำหรับ V7 (Work Breakdown)

### Phase 1: รายงานการเช็คชื่อ (Reports Export)

#### 3.1 PDF Export Module
- [ ] ติดตั้งฟอนต์ภาษาไทย (เช่น THSarabunNew หรือฟอนต์สารบรรณรุ่นอื่น) ในรูปแบบ Base64 และลงทะเบียนฟอนต์ใน `jsPDF` เพื่อให้พิมพ์ภาษาไทยได้จริง
- [ ] พัฒนา Template หัวกระดาษรายงาน และจัดตารางสรุปรายงานการมาเรียน (มา, ขาด, สาย, ลา) ให้สวยงาม รองรับรายงานแบบแบ่งหน้า (Multi-page PDF)

#### 3.2 CSV Export Module
- [ ] ปรับปรุงการส่งออก CSV โดยใช้ BOM (Byte Order Mark) `\ufeff` นำหน้าเนื้อหา เพื่อบังคับให้ Microsoft Excel อ่านข้อมูลภาษาไทยเป็น UTF-8 โดยไม่ต้องแปลงค่าซ้ำซ้อน

---

### Phase 2: Supabase Security (RLS Rollout)

#### 3.3 การบังคับใช้สิทธิ์ (RLS Implementation Steps)
- [ ] อัปเดตไฟล์ SQL Migrations เพื่อสั่งเปิด RLS ทีละตารางตามลำดับความเสี่ยง
- [ ] เขียน SQL RLS Policy สำหรับ:
  1. `students` และ `classrooms`
  2. `student_group_memberships` (สิทธิ์เขียนเฉพาะ ADMIN, อ่านเฉพาะครู)
  3. `attendance` และ `attendance_sessions` (ครูเขียนประวัติได้เฉพาะวิชาที่อยู่ในตารางสอนของตนเอง)
