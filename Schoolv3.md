# Development Plan: School Attendance Management System (SAMS) - Version 3
## แผนการพัฒนาและประวัติการอัปเดตระบบ (อ้างอิงจาก Schoolv2.md)

ไฟล์นี้เป็นจุด Checkpoint สำหรับอ้างอิงสถานะล่าสุดของโปรเจกต์ เพื่อให้ AI หรือนักพัฒนาคนต่อไปสามารถอ่านและทำงานต่อยอดได้อย่างราบรื่น

---

### 🟢 สรุปฟีเจอร์ที่เพิ่งพัฒนาเสร็จสิ้น (Recent Accomplishments in V3)
1. **Responsive UI & Navigation**:
    - ปรับปรุงแถบเมนู (Sidebar) ให้รองรับการใช้งานผ่านมือถือ (Mobile-friendly) มีระบบ Overlay และพับเก็บอัตโนมัติ
    - จัดเรียงลำดับเมนูใหม่ตาม Workflow การทำงานจริงของครู (เอาเมนูที่ใช้บ่อยไว้ด้านบน)
    - เพิ่ม Custom Logout Confirmation Modal ป้องกันการกดออกจากระบบโดยไม่ตั้งใจ
2. **Role-Based Access Control (RBAC)**:
    - จำกัดสิทธิ์การมองเห็นเมนูตาม Role (ADMIN เห็นทั้งหมด, TEACHER เห็นเฉพาะเมนูจัดการนักเรียน, เช็คชื่อ และรายงาน)
    - เพิ่มหน้า "User Management" ในเมนูตั้งค่าระบบ (Settings) ให้ Super Admin สามารถจัดการ Role ของผู้ใช้ได้
3. **Analytics & Reports (ระบบรายงานผล)**:
    - สร้างหน้า "รายงานสรุป" (Reports) พร้อม UI กราฟสถิติที่สวยงาม (Recharts)
    - ดึงข้อมูลจริงจากฐานข้อมูลมาแสดงผล ทั้งจำนวนนักเรียน กราฟแท่ง (Bar Chart) และกราฟวงกลม (Pie Chart)
    - เพิ่มปุ่มกรองช่วงเวลา (วันนี้, สัปดาห์นี้, เดือนนี้, เทอมนี้, ทั้งปี) ซึ่งทำงานตัดข้อมูลในฐานข้อมูลได้จริง
    - ทำ Shortcut จาก Executive Dashboard กล่อง "รายวิชาในระบบ" ให้ลิงก์มายังแท็บรายงานรายวิชา
4. **Data Management Enhancements**:
    - แก้ไขบั๊ก SQL RPC `create_teacher_account` ที่ทำให้ระบบ Auth Error 500 (แก้โดยการใส่ค่า confirmation_token ให้ครบถ้วน)
    - เพิ่มตัวกรองห้องเรียน (Classroom Filter) ในหน้าจัดการข้อมูลนักเรียน เพื่อไม่ให้รายการแสดงผลยาวเกินไป
5. **UI Cleanup**:
    - ซ่อนเมนู "ระบบการลา" (Leave System) ออกจากแถบเมนูด้านซ้ายชั่วคราวตามที่ระบุ (รอการพัฒนาต่อในอนาคต)

### 🟡 สถานะการพัฒนาแบ่งตาม Phase (ปัจจุบัน)

- [x] Database & Backend Setup (Supabase, Tables, RLS)
- [x] Authentication Module & RBAC

- [x] แก้ไขบั๊กการเพิ่มบัญชีครู (RPC Fix)

#### Phase 2: Scheduling & Basic Attendance (ระบบตารางเรียนและการเช็คชื่อพื้นฐาน)
*สถานะ: เกือบสมบูรณ์ (95%)*
- [x] ระบบจัดตารางเรียน / ตารางสอน
- [x] หน้าจอสำหรับครูที่ปรึกษาในการเช็คชื่อเข้าแถว (Homeroom)
- [x] หน้าจอเช็คชื่อนักเรียนรายคาบ (Attendance)
- [x] Executive Dashboard สำหรับผู้บริหาร
- [ ] *สิ่งที่ต้องทำต่อ: ปรับปรุงการตรวจสอบการจัดตารางชนกันให้รัดกุมขึ้น*

#### Phase 3: Smart Attendance & Anti-Cheat (ระบบเช็คชื่ออัจฉริยะ)
*สถานะ: เลื่อนการพัฒนาออกไปก่อนชั่วคราวตามคำสั่ง (Deferred)*

#### Phase 4: Leave Management & Notifications (ระบบการลาและการแจ้งเตือน)
*สถานะ: ระงับชั่วคราว (Paused)*

#### Phase 5: Reporting, Analytics & Polish (ระบบรายงานผลและการปรับปรุงขั้นสุดท้าย)
*สถานะ: กำลังดำเนินการ (In Progress)*

- [ ] *สิ่งที่ต้องทำต่อ 1: ดึงข้อมูลและสร้างตารางสถิติสำหรับแท็บ "รายห้องเรียน", "รายบุคคล", และ "รายวิชา" ในหน้า Reports.tsx*
