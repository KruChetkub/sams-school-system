เอกสารฉบับนี้จัดทำขึ้นเพื่อให้ครูผู้ดูแลระบบ CKW สามารถติดตั้งระบบสำรองข้อมูลอัตโนมัติ และเรียนรู้วิธีกู้คืนฐานข้อมูลหากเกิดกรณีฉุกเฉินในอนาคตได้อย่างถูกต้อง

## 🛠️ ส่วนที่ 1: การติดตั้ง Google Apps Script (ระบบอัตโนมัติรายวัน)

เพื่อให้ระบบบันทึก Log และระบบสำรองข้อมูลทำหน้าที่ **ย้าย Log ประจำวัน** และ **สำรองฐานข้อมูลถาวร** คุณครูสามารถเปิดใช้บริการสคริปต์ได้ฟรีบนบัญชี Google ส่วนบุคคลดังนี้:

### ขั้นตอนการสร้างสคริปต์:
1. เข้าเว็บไซต์ [Google Apps Script Dashboard](https://script.google.com)
2. คลิกปุ่ม **"โครงการใหม่" (New Project)** ทางด้านซ้ายบน
3. ตั้งชื่อโครงการด้านบนสุดว่า: `SAMS_Backup_And_Logs`
4. ลบโค้ดเริ่มต้นทั้งหมดในโปรแกรมแก้ไขข้อความ และคั่นด้วยการเปิดไฟล์ [gas_sams_v12.js](file:///d:/sams-school-system-main/frontend/google-apps-script/gas_sams_v12.js) คัดลอกโค้ดทั้งหมดมาวาง
5. ปรับเปลี่ยนการตั้งค่าที่หัวข้อ **CONFIGURATION** ด้านบนสุดของโค้ด:
   - `SUPABASE_URL`: เปลี่ยนให้ตรงกับโปรเจกต์ Supabase ของโรงเรียน
   - `SUPABASE_KEY`: นำคีย์ `service_role` (คีย์ลับระดับผู้จัดการ) หรือ `anon` มาวาง
   - `NOTIFICATION_EMAIL`: กรอกอีเมลส่วนตัวของคุณครูเพื่อรับการแจ้งเตือนหากระบบทำงานล้มเหลว/สำเร็จ
   - `LINE_NOTIFY_TOKEN`: กรอก Token ของ LINE Notify หากต้องการให้แจ้งเตือนผลลัพธ์ผ่าน LINE
   - `TELEGRAM_BOT_TOKEN` & `TELEGRAM_CHAT_ID`: กรอก Token และ Chat ID ของบอต Telegram หากต้องการแจ้งเตือนผลลัพธ์ผ่าน Telegram
6. กดปุ่ม 💾 **บันทึกโครงการ (Save Project)**

### ขั้นตอนการตั้งเวลารันอัตโนมัติ (Triggers):
เพื่อให้โค้ดดังกล่าวทำงานด้วยตนเองทุกๆ วันตอนดึก ให้ตั้งค่าทริกเกอร์ดังนี้:
1. ที่แถบเมนูด้านซ้าย คลิกไอคอน ⏰ **ทริกเกอร์ (Triggers)**
2. คลิกปุ่ม **"+ เพิ่มทริกเกอร์" (+ Add Trigger)** ด้านล่างขวา
3. **ตั้งค่าทริกเกอร์ที่ 1 (ระบบโยกย้าย Log):**
   - *เลือกฟังก์ชันที่จะเรียกใช้:* `runDailyLogOffload`
   - *เลือกเวอร์ชันที่จะเรียกใช้:* `Head`
   - *เลือกแหล่งที่มาของเหตุการณ์:* `ตามเวลา` (Time-driven)
   - *เลือกประเภททริกเกอร์ตามเวลา:* `เครื่องจับเวลารายวัน` (Day timer)
   - *เลือกเวลา:* `01:00 น. ถึง 02:00 น.` (ตีหนึ่ง ถึง ตีสอง)
   - คลิก **บันทึก (Save)** และกดยอมรับสิทธิ์การเข้าถึงความปลอดภัย (Authorize) ของบัญชี Google ของคุณครู
4. **ตั้งค่าทริกเกอร์ที่ 2 (ระบบสำรองฐานข้อมูลหลัก):**
   - คลิก **"+ เพิ่มทริกเกอร์"** อีกครั้ง
   - *เลือกฟังก์ชันที่จะเรียกใช้:* `runDailyDatabaseBackup`
   - *เลือกประเภททริกเกอร์ตามเวลา:* `เครื่องจับเวลารายวัน` (Day timer)
   - *เลือกเวลา:* `02:00 น. ถึง 03:00 น.` (ตีสอง ถึง ตีสาม)
   - คลิก **บันทึก (Save)**

*เมื่อทำครบถ้วน Google จะเริ่มจัดระเบียบและคัดลอกข้อมูลไปเก็บใน Google Drive ของคุณครูทุกคืนอัตโนมัติครับ!*

## 💾 ส่วนที่ 2: วิธีการกู้คืนฐานข้อมูล (Data Restoration)

หากฐานข้อมูลบน Supabase เกิดปัญหา หรือมีการลบข้อมูลนักเรียน/การเช็คชื่อที่ผิดพลาด คุณครูสามารถนำไฟล์ `.json` ที่สำรองไว้บน Google Drive ในโฟลเดอร์ `01_Database_JSON` มานำเข้ากลับหลังบ้านได้ทันทีโดยใช้เครื่องมือ [restore.js](file:///d:/sams-school-system-main/frontend/google-apps-script/restore.js) ที่ผมเตรียมไว้ให้แล้ว:

### ⚙️ สิ่งที่ต้องเตรียมก่อนกู้คืน (Prerequisites):
1. **เพิ่มคีย์ระดับจัดการในไฟล์ `.env`:**
   เปิดไฟล์ `.env` ในโฟลเดอร์หลัก `d:/sams-school-system-main/frontend/.env` แล้วเพิ่มตัวแปรด้านล่างนี้ลงไป (หากมีแล้วให้ตรวจสอบความถูกต้อง):
   ```env
   # คีย์ระดับจัดการสูง (Service Role Key) ของ Supabase เพื่อใช้เขียนข้อมูลข้าม RLS
   SUPABASE_SERVICE_ROLE_KEY=sb_secret_xxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   *(สามารถหาคีย์นี้ได้จากแถบตั้งค่าในเว็บ Supabase -> Project Settings -> API -> คีย์ชื่อ `service_role`)*

### 📝 ขั้นตอนการกู้คืนข้อมูล:
1. เปิดโปรแกรม Command Prompt (CMD) หรือ PowerShell ในคอมพิวเตอร์ของคุณ
2. เข้าไปยังห้องเก็บสคริปต์ด้วยคำสั่ง:
   ```bash
   cd d:\sams-school-system-main\frontend\google-apps-script
   ```
3. ติดตั้งโปรแกรมสนับสนุน (เฉพาะการทำครั้งแรกสุด):
   ```bash
   npm install @supabase/supabase-js dotenv
   ```
4. ดาวน์โหลดไฟล์ข้อมูลสำรองที่เราต้องการจาก Google Drive (เช่น `db_backup_20260531.json`) และนำมาวางไว้ในโฟลเดอร์ `d:\sams-school-system-main\frontend\google-apps-script`
5. รันคำสั่งเริ่มกระบวนการเขียนข้อมูลกลับสู่ Supabase:
   ```bash
   node restore.js db_backup_20260531.json
   ```
6. หน้าจอจะแสดงการนำเข้าเรียงลำดับตารางอย่างเป็นขั้นตอน และขึ้นข้อความสำเร็จดังนี้:
   ```text
   ✅ กู้คืนตาราง students สำเร็จ
   ✅ กู้คืนตาราง attendance สำเร็จ
   🎉 สำเร็จ! กู้คืนฐานข้อมูลระบบ SAMS เป็นเวสชั่น 12 เรียบร้อยแล้ว
   ```

> [!CAUTION]
> **ข้อควรระวัง:** การกู้คืนระบบผ่านการเขียนข้อมูลทับ (Upsert) จะทำให้ข้อมูลการเปลี่ยนแปลงระหว่างวันหรือหลังจากวันที่สำรองข้อมูลนั้นถูกแก้ไขกลับไปเป็นสถานะเดิมทันที แนะนำให้กู้คืนเฉพาะยามจำเป็นอย่างยิ่งเท่านั้น!
