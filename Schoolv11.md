# Development Plan: School Attendance Management System (SAMS) - Version 11
## สรุปการอัปเดตล่าสุด + Baseline สำหรับพัฒนาต่อจาก V10 (โฟกัสความถูกต้องของแบบฟอร์มและการพิมพ์รายงาน)

เอกสารนี้สรุปงานที่พัฒนาเพิ่มเติมในรอบล่าสุด เพื่อใช้เป็น baseline ของ V11 โดยเน้นการเพิ่มประสิทธิภาพ ความถูกต้องในการแมปข้อมูล และการแสดงชื่อผู้ใช้งานจริงในระบบลงบนลายเซ็นผู้รับรอง

---

## 1) สถานะความสำเร็จในรอบล่าสุด (V11 Completed)

### 1.1 การปรับปรุงระบบพิมพ์รายงาน (ReportPrintView & PDF Layout)
- [x] **การจัดตารางขอบเขต A4:** ปรับโครงสร้าง CSS ใน `ReportPrintView.tsx` ตั้งค่า margin เป็น `20mm` ซ้ายขวา และ `12mm` บนล่าง ทำให้ Layout การแสดงผลหน้ากระดาษ A4 สวยงามเป็นระเบียบ ไม่ล้นหน้า
- [x] **การสเกลรูปภาพถ่ายบ้านนักเรียน:** ขยายการสเกลรูปภาพทั้ง 2 ภาพ (ภายนอกและภายในบ้าน) ในหน้าพิมพ์รายงาน 3/4 ให้มีขนาดพอดีหน้ากระดาษ สวยงาม ชัดเจน และลดช่องว่างสีขาวที่เหลืออยู่
- [x] **การซ่อนหน้าพิมพ์บนแท็บกรอกข้อมูล:** กำหนดให้ส่วนประกอบ `<ReportPrintView>` ซ่อนตัวอย่างสมบูรณ์เมื่ออยู่บนขั้นตอนแบบฟอร์มแท็บที่ 1 ถึง 4 และจะปรากฏเฉพาะในขั้นตอนที่ 5 (แสดงรายงาน) เพื่อเพิ่มความเสถียรของหน้ากรอกข้อมูล
- [x] **การกระชับเนื้อหาหน้าแรก:** ขยับจัดแต่งระยะห่างบรรทัดเนื้อหาในหน้ารายงานแผ่นที่ 1 ให้กระชับขึ้น เพื่อให้เนื้อหาแสดงลงมาถึงข้อ **4.2 (พฤติกรรมและความเสี่ยงในห้องเรียน)** ได้อย่างพอดี ไม่ล้นไปหน้าสอง
- [x] **การจัดแนวย่อหน้าหัวข้อย่อย (3.1 ถึง 4.12):** เพิ่มคลาส `.sub-section { padding-left: 4.5mm; }` ใน CSS ของ `ReportPrintView.tsx` และปรับปรุงแท็กครอบหัวข้อย่อยทั้งหมดตั้งแต่ข้อ `3.1, 3.3 - 3.8` และ `4.1 - 4.12` เพื่อให้หลักแรกของตัวเลขหัวข้อย่อยขยับเยื้องตรงกันกับตัวอักษรตัวแรกของหัวข้อหลัก (เช่น ตัว "ค" ในข้อ 3. ความสัมพันธ์ในครอบครัว และตัว "พ" ในข้อ 4. พฤติกรรมและความเสี่ยง) ได้อย่างสวยงามสมบูรณ์แบบในการพิมพ์รายงาน


### 1.2 การแก้ไขความถูกต้องของการดึงข้อมูลและ Mapping ฟิลด์ (Data Correctness)
- [x] **แก้ไขระบบตรวจจับตัวเลือก 3.3 ถึง 3.7:** อัปเกรดการเทียบค่า String ให้ตรงกับฐานข้อมูลเป๊ะๆ ช่วยให้การแสดงตัวเลือกเรื่องการอยู่คนเดียว รายได้เฉลี่ย แหล่งรายได้ และความช่วยเหลือของโรงเรียนทำงานได้เสถียร
- [x] **แก้ไขข้อ 4.11 (การเข้าถึงคอมพิวเตอร์และอินเทอร์เน็ตที่บ้าน):** แก้ไขความคลาดเคลื่อนของคำ โดยที่หน้าบันทึกฟอร์มใช้คำว่า *"สามารถเข้าถึง Internet..."* (มีคำว่า "ถึง") แต่ในหน้าพิมพ์ใช้คำว่า *"สามารถเข้า Internet..."* (ไม่มีคำว่า "ถึง")
  - *วิธีแก้:* ปรับฟังก์ชันเทียบคำใน `ReportPrintView.tsx` ให้ตรวจจับทั้งสองแบบ (รวมถึงรองรับคำว่า "สามารถเข้าถึงได้" / "ไม่สามารถเข้าถึงได้") ทำให้เครื่องหมายถูก `✓` แสดงผลตามที่ผู้ใช้กรอกจริง 100%

### 1.3 ดึงข้อมูลครูผู้ใช้งานปัจจุบัน (Logged-in Teacher Signature System)
- [x] **แก้ไขปัญหาการ Import Supabase:** นำเข้า `import { supabase } from '../../lib/supabase'` ลงใน `VisitForm.tsx` อย่างถูกต้อง แก้ไขข้อผิดพลาดเงียบ (Silent Reference Error) ที่ทำให้การคิวรีโปรไฟล์ไม่สำเร็จ
- [x] **เชื่อมโยงข้อมูลครูในหน้ากรอกข้อมูล (แท็บ 3):** บนฟอร์มหน้าแท็บที่ 3 (ภาพถ่าย) บริเวณรับรองภาพถ่ายบ้านนักเรียน จากเดิมที่เป็นจุดไข่ปลาเปล่า `(......................................................)` ได้ปรับแก้ให้แสดงชื่อ-นามสกุลจริงของครูผู้ใช้งานระบบที่ล็อกอินอยู่ปัจจุบันทันที (เช่น **นางสาวภัทร์ธิชวี ปันใจ**) เพื่อแสดงตัวตนผู้บันทึกอย่างชัดเจน
- [x] **เชื่อมโยงข้อมูลครูในหน้าพิมพ์รายงาน (หน้า 3/4):** เปลี่ยนจากเดิมที่ใช้ครูประจำชั้น (Advisor) ให้มาเป็นชื่อของครูผู้เยี่ยมบ้านและบันทึกข้อมูลตัวจริงที่ล็อกอินเข้าสู่ระบบ (`currentUserFullName`) ในส่วนตัวแปร `teacherName` ทำให้ข้อมูลลายเซ็นรับรองภาพถ่ายบ้านทั้งสองหน้าสอดคล้องถูกต้องตรงกัน

*ไฟล์ที่แก้:* `src/pages/homevisit/VisitForm.tsx`, `src/components/homevisit/ReportPrintView.tsx`

## 2) Database Schema Constraints (V11 Reminders)

- **ตาราง `teachers`:** ไม่มีฟิลด์ `prefix` (คำนำหน้าชื่อ) ดังนั้นการคิวรีหรือ Query ข้อมูลใดๆ เพื่อนำชื่อคุณครูมาแสดง จะต้องดึงเฉพาะฟิลด์ `first_name` และ `last_name` เท่านั้นเพื่อหลีกเลี่ยงข้อผิดพลาด `400 Bad Request` จากฐานข้อมูล

## 3) ข้อเสนอแนะสำหรับการพัฒนาต่อในเวอร์ชันถัดไป (Future Work / V12)

- **การ Export ข้อมูลแบบ Multi-format:** พัฒนาปุ่มคลิกเดียวเพื่อดาวน์โหลดรายงานเยี่ยมบ้านทั้งหมดเป็นไฟล์ PDF รวม หรือดึงข้อมูลสรุปแบบประเมินความเสี่ยงส่งออกเป็น Excel (.xlsx)
- **ระบบยืนยันตนแบบ Multi-teacher homeroom:** รองรับห้องเรียนที่มีครูประจำชั้นร่วมกัน 2 คน เพื่อให้สิทธิ์ทั้งคู่ในการเซ็นรับรองหรือเข้าดูประวัติร่วมกัน
- **ระบบลากและระบุพิกัดบนแผนที่ด้วยมือ (Manual Coordinates Adjustment):** ในกรณีที่ GPS อุปกรณ์เคลื่อนคลาดเคลื่อน ให้คุณครูสามารถลาก Pin ไปวางตรงบ้านที่แท้จริงได้โดยตรงบนหน้าจอแบบฟอร์ม
